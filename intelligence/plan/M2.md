## Target State (TO-BE)

### New CLI option

```
reponova build --start-after <phase>
```

**Semantics**: Skip the named phase and all its ancestors. Execute only the strict **descendants** (phases that transitively depend on the named phase).

**Validation**:
1. Phase must exist in the DAG
2. Phase's outputs must exist on disk (verified by checking output files presence)
3. Mutually exclusive with `--target` (yargs `.conflicts("start-after", "target")`)

**Examples** (new DAG: 8 phases, 5 levels):

```
reponova build --start-after enrich
  → Runs: index, embeddings, html, report (Level 4)
  → Skips: file-detection, graph, outlines, communities, enrich (Levels 0-3)

reponova build --start-after communities
  → Runs: enrich, index, embeddings, html, report (Levels 3-4)
  → Skips: file-detection, graph, outlines, communities (Levels 0-2)

reponova build --start-after graph
  → Runs: communities, enrich, index, embeddings, html, report (Levels 2-4)
  → Skips: file-detection, graph, outlines (Levels 0-1)
  → NOTE: outlines has no descendants in the default DAG, so it's excluded

reponova build --start-after file-detection
  → Runs: graph, outlines, communities, enrich, index, embeddings, html, report (Levels 1-4)
  → Skips: file-detection (Level 0)
```

## Implementation Steps

### Step 1: Add `resolveTransitiveDescendants` to DAG utilities

**File: `src/pipeline/engine/dag.ts`**

New function — the inverse of `resolveTransitiveDeps`:

```typescript
/**
 * Resolve all strict descendants of a phase.
 * Returns phases that transitively depend on the given phase (NOT including the phase itself).
 */
export function resolveTransitiveDescendants(
  dag: Map<string, Phase>,
  sourceId: string,
): Set<string> {
  if (!dag.has(sourceId)) {
    throw new Error(
      `Phase "${sourceId}" not found. Available: ${[...dag.keys()].join(", ")}`
    );
  }

  const result = new Set<string>();

  // Build reverse adjacency: for each phase, which phases depend on it?
  const dependents = new Map<string, Set<string>>();
  for (const [id, phase] of dag) {
    for (const dep of phase.dependencies) {
      if (!dependents.has(dep)) dependents.set(dep, new Set());
      dependents.get(dep)!.add(id);
    }
  }

  // BFS from sourceId through dependents
  const queue = [sourceId];
  while (queue.length > 0) {
    const current = queue.shift()!;
    const children = dependents.get(current);
    if (!children) continue;
    for (const child of children) {
      if (!result.has(child)) {
        result.add(child);
        queue.push(child);
      }
    }
  }

  return result;
}
```

### Step 2: Add output existence validation

**File: `src/pipeline/engine/dag.ts`** (or new file `src/pipeline/engine/phase-outputs.ts`)

We need a registry of expected output files per phase. This allows `--start-after` to validate that the named phase has already produced its outputs.

 * Used by --start-after to validate prerequisite outputs exist.
 */

  "file-detection": ["detected-files.json"],
  "graph": ["graph-nodes.json"],
  "outlines": [],  // outlines/ dir — checked separately
  "communities": ["graph.json"],
  "enrich": ["graph-enriched.json", "node_descriptions.json", "community_summaries.json"],
  "index": ["graph_search.db"],
  "embeddings": [],  // vectors/ dir — checked separately
  "html": ["graph.html"],
  "report": ["report.md"],
};

/**
 * Known output directories for each phase (relative to outputDir).
 */
export const PHASE_OUTPUT_DIRS: Record<string, string[]> = {
  "outlines": ["outlines"],
  "embeddings": ["vectors"],
};
```

Validation function:

```typescript
import { existsSync } from "node:fs";
import { join } from "node:path";

/**
 * Check that a phase's expected outputs exist on disk.
 * Throws if any required output is missing.
 */
export function validatePhaseOutputsExist(
  phaseId: string,
  outputDir: string,
): void {
  const files = PHASE_OUTPUTS[phaseId] ?? [];
  const dirs = PHASE_OUTPUT_DIRS[phaseId] ?? [];

  const missing: string[] = [];

  for (const file of files) {
    if (!existsSync(join(outputDir, file))) {
      missing.push(file);
    }
  }

  for (const dir of dirs) {
    if (!existsSync(join(outputDir, dir))) {
      missing.push(`${dir}/`);
    }
  }

  if (missing.length > 0) {
    throw new Error(
      `Cannot use --start-after "${phaseId}": missing output files: ${missing.join(", ")}. ` +
      `Run a full build first, or run with --target to include this phase.`
    );
  }
}
```

### Step 3: Update `OrchestratorOptions` and orchestration logic

**File: `src/pipeline/engine/orchestrator.ts`**

```diff
 export interface OrchestratorOptions {
   /** Run only this phase + its transitive deps (null = full DAG) */
   target?: string;
+  /** Run only strict descendants of this phase (null = full DAG) */
+  startAfter?: string;
   /** Force all phases to ignore cache */
   force: boolean;
   /** Max concurrent phases per level (0 = unlimited) */
   concurrency?: number;
 }
```

In `orchestrate()`:

```typescript
// Prune to target + transitive deps if --target specified
if (options.target) {
  const keep = resolveTransitiveDeps(dag, options.target);
  dag = pruneDAG(dag, keep);
  log.info(`Target: ${options.target} (${keep.size} phases in dependency chain)`);
}

// Prune to strict descendants if --start-after specified
if (options.startAfter) {
  validatePhaseOutputsExist(options.startAfter, ctx.outputDir);
  const descendants = resolveTransitiveDescendants(dag, options.startAfter);
  if (descendants.size === 0) {
    log.info(`No phases downstream of "${options.startAfter}" — nothing to run.`);
    return { outputDir: ctx.outputDir, phases: new Map(), totalProcessed: 0 };
  }
  dag = pruneDAG(dag, descendants);
  log.info(`Start after: ${options.startAfter} (${descendants.size} downstream phases)`);
}
```

**Important**: When the DAG is pruned to only descendants, those descendants still list the source phase in their `dependencies` array. But after pruning, that dependency is no longer in the DAG. The `validate()` function would fail.

**Solution**: Call `validate(dag)` BEFORE pruning (already done). After pruning for `--start-after`, do NOT re-validate — the pruned DAG intentionally has dangling dependencies pointing to the excluded ancestor. `topologicalLevels()` handles this correctly because it only traverses deps that exist in the DAG:

Actually, looking at `topologicalLevels()`:

```typescript
function getLevel(id: string): number {
  const phase = dag.get(id)!;
  if (phase.dependencies.length === 0) {
    assigned.set(id, 0);
    return 0;
  }
  let maxDepLevel = -1;
  for (const dep of phase.dependencies) {
    maxDepLevel = Math.max(maxDepLevel, getLevel(dep));
  }
  ...
}
```

This calls `dag.get(dep)!` which would crash if `dep` is not in the pruned DAG.

**Fix**: Modify `topologicalLevels()` to skip dependencies not present in the DAG (treat as satisfied):

```typescript
function getLevel(id: string): number {
  if (assigned.has(id)) return assigned.get(id)!;

  const phase = dag.get(id)!;
  let maxDepLevel = -1;
  for (const dep of phase.dependencies) {
    // Skip dependencies not in the current (potentially pruned) DAG
    if (!dag.has(dep)) continue;
    maxDepLevel = Math.max(maxDepLevel, getLevel(dep));
  }

  const level = maxDepLevel + 1;
  assigned.set(id, level);
  return level;
}
```

This change is safe for the existing `--target` behavior too, since `--target` pruning keeps all ancestors (so deps are always present). The `if (!dag.has(dep)) continue` is a no-op for `--target` — only activates for `--start-after`.

describe("build --start-after E2E", { timeout: 60000 }, () => {
  let tmpDir: string;

  beforeEach(() => {
    tmpDir = mkdtempSync(join(tmpdir(), "reponova-start-after-e2e-"));
    writeFile(tmpDir, "main.py", "def greet():\n    '''Hello'''\n    pass\n");
    writeFile(tmpDir, "utils.py", "def helper():\n    greet()\n");
  });

  afterEach(() => {
    rmSync(tmpDir, { recursive: true, force: true });
  });

  it("--start-after enrich runs only Level 4 phases", async () => {
    // Full build first
    await runBuild(makeConfig(), tmpDir, {});
    // Then start-after
    const result = await runBuild(makeConfig(), tmpDir, { startAfter: "enrich" });

    expect(result.phases.has("file-detection")).toBe(false);
    expect(result.phases.has("graph")).toBe(false);
    expect(result.phases.has("communities")).toBe(false);
    expect(result.phases.has("enrich")).toBe(false);
    expect(result.phases.has("index")).toBe(true);
    expect(result.phases.has("report")).toBe(true);
  });

  it("--start-after communities runs enrich + Level 4", async () => {
    await runBuild(makeConfig(), tmpDir, {});
    const result = await runBuild(makeConfig(), tmpDir, { startAfter: "communities" });

    expect(result.phases.has("communities")).toBe(false);
    expect(result.phases.has("enrich")).toBe(true);
    expect(result.phases.has("index")).toBe(true);
  });

  it("--start-after fails when outputs don't exist", async () => {
    // No prior build — outputs missing
    await expect(
      runBuild(makeConfig(), tmpDir, { startAfter: "enrich" })
    ).rejects.toThrow(/missing output files/);
  });

  it("--start-after fails for unknown phase", async () => {
    await expect(
      runBuild(makeConfig(), tmpDir, { startAfter: "nonexistent" })
    ).rejects.toThrow(/not found/);
  });

  it("--start-after with --force re-runs downstream phases", async () => {
    await runBuild(makeConfig(), tmpDir, {});
    const result = await runBuild(makeConfig(), tmpDir, { startAfter: "enrich", force: true });

    // Phases ran (not skipped by cache)
    const indexResult = result.phases.get("index");
    expect(indexResult?.skipped).toBe(false);
  });

  it("--start-after for leaf phase (report) results in empty run", async () => {
    await runBuild(makeConfig(), tmpDir, {});
    const result = await runBuild(makeConfig(), tmpDir, { startAfter: "report" });

    expect(result.phases.size).toBe(0);
    expect(result.totalProcessed).toBe(0);
  });

  it("--start-after graph excludes outlines (no descendants of outlines via graph)", async () => {
    const config = makeConfig();
    config.outlines.enabled = true;
    await runBuild(config, tmpDir, {});
    const result = await runBuild(config, tmpDir, { startAfter: "graph" });

    // outlines depends on file-detection, not graph → NOT a descendant of graph

    expect(result.phases.has("enrich")).toBe(true);
  });

## README Changes

Add to CLI Reference under `reponova build`:

```markdown
| `--start-after` | No | Run only phases downstream of this phase. Requires a prior build with outputs present. Mutually exclusive with `--target`. |
```

Add usage examples:

```markdown
**Start-after examples:**

\```bash
reponova build --start-after enrich      # re-run index + embeddings + html + report only
reponova build --start-after communities # re-run enrich + Level 4 phases
reponova build --start-after graph       # re-run communities + enrich + Level 4
\```

Use `--start-after` when you've made changes to a phase's configuration but don't need to re-run upstream phases. For example, after changing embedding settings, run `--start-after enrich` to regenerate embeddings without re-parsing files or re-computing communities.
```
